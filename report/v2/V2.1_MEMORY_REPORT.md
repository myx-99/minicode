# Claude Code Mini — V2.1 跨轮次记忆重写报告

> **项目**: Claude Code Mini V2.1 — Cross-Turn Memory Rewrite
> **基线**: V2 (221 tests, 双模式, ContextManager)
> **报告日期**: 2026-06-03

---

## 1. 问题回顾

V2 实测暴露三类问题：

| # | 现象 | 根因 |
|---|------|------|
| B1 | REPL 第二轮 Task「删除刚刚完成的任务」，Agent 不知道「刚刚」指什么 | 每轮 `stream(task)` 创建全新 graph，无 Session 级 state |
| B2 | 第三轮「刚才完成了什么」，Agent 说无法访问对话历史 | `LongTermMemory.search(当前task)` 对 meta 问题召回失败 |
| B3 | `.agent/memory/entries.jsonl` 写入摘要为空 | `finish_node` 在 `final_answer` 赋值**之前**调用 `summarize_session` |

---

## 2. 解决方案

### 2.1 新架构：两层记忆 + 统一管理器

```
┌─────────────────────────────────────────────────────────┐
│                    MemoryManager                         │
│  ┌──────────────────┐    ┌──────────────────────────┐  │
│  │  SessionMemory    │    │  ProjectMemory            │  │
│  │  REPL 跨轮次       │    │  跨进程持久化              │  │
│  │  (当前会话 N 轮)   │    │  (turns.jsonl)           │  │
│  └────────┬─────────┘    └────────────┬─────────────┘  │
│           └──────────┬────────────────┘                 │
│                      ▼                                    │
│           build_context_for_task(task)                  │
│           record_completed_turn(state, final_answer)      │
└─────────────────────────────────────────────────────────┘
```

### 2.2 核心设计决策

| V2 (旧) | V2.1 (新) |
|---------|----------|
| `MemoryEntry` + `category` 碎片模型 | `TurnRecord` 完整任务记录 |
| `LongTermMemoryKeyword` 统一类 | `SessionMemory` + `ProjectMemory` + `MemoryManager` 三层分离 |
| `finish_node` 调用 `summarize_session()` | `finish_node` 调用 `record_completed_turn(state, final_answer)` |
| 关键词检索 + category 权重 | 关键词 Jaccard + meta query 检测 + 始终注入 recent turns |
| `memory_context` 作为 init 注入 | `session_context` 合并 session + project 记忆 |

### 2.3 B3 修复（finish_node 写入顺序）

```
旧:                                                                               新:
finish_node                                                                       finish_node
  ├── summarize_session(state)   ← final_answer 还未生成！                           ├── 生成 final_answer via LLM
  ├── 生成 final_answer                                                             ├── final_answer 已就绪
  └── return                                                                        ├── record_completed_turn(state, final_answer)  ✅
                                                                                    └── return
```

### 2.4 Meta Query 检测

```
META_PATTERNS = [r"刚才", r"刚刚", r"上一个", r"之前", r"上次",
                 r"what did i", r"what was", r"previous task", r"last task",
                 r"完成了什么", r"做了什么", r"删.*刚刚"]
```

Meta query 时：加大 session turns 数量到 2×，跳过 keyword search（它没用），确保 Agent 能看到之前做了什么。
